1단계 · 차트 선정

renderStep1(signal) · 메타데이터 출력
분석 대상의 기본 정보를 확인하는 단계. 별도 계산 없이 signal 객체의 메타데이터를 포맷팅하여 출력한다.
입력 필드 (signal 객체)
  • signal.pair페어명 (예: BTCUSDT)
  • signal.timestamp분석 시각 Unix timestamp (없으면 signal.analysisDate 사용)
  • signal.currentPrice현재가 (fmtPrice 함수로 포맷)
출력 예시
## 1단계 · 차트 선정 - 페어: BTCUSDT - 분석 TF: HTF 4H / LTF 15M - 분석 시각: 2026-05-15 14:00:00Z - 현재가: $93,540.0
fmtPrice 규칙: $1,000 이상이면 $93,540.0, 미만이면 소수점 4자리 후 trailing zero 제거 (예: $0.0843)

2단계 · 구조 분석

renderStep2(signal) · HTF/LTF 추세 정렬 + AMD 단계 판단
멀티 타임프레임(HTF 4H / LTF 15M) 추세가 일치하는지 확인하고, 기관 주기(AMD)의 현재 단계를 출력한다. 이 단계에서 HTF ≠ LTF면 충돌 경고를 발생시켜 진입 트리거 부재를 명시한다.
추세 정렬 로직
const aligned = htf === ltf ? 'HTF와 정렬' : 'HTF와 충돌';

// htf !== ltf 이면 충돌 텍스트 추가:
// "HTF 상승 추세 + LTF 하락 추세 → 진입 트리거 부재"

두 추세가 일치해야 진입 후보. 충돌 시 관망.

AMD 단계 감지 로직 (amd.js)
단계조건
DISTRIBUTION스윕 확정 + 이후 BOS 발생 (반대 방향)
MANIPULATION최근 10봉 내 확정 스윕, BOS 없음
ACCUMULATION좁은 스윙 범위 + 스윕/BOS 없음
RESET위 조건 미해당 (기본값)
AMD 판단 상세 (detectAMDPhase)
// 1. 최근 20봉 평균 범위 계산 → avgRange
// 2. 최근 스윙 10개의 고저 범위 → swingRange

// DISTRIBUTION: 확정 스윕 + 스윕 이후 BOS + 반대 방향
if (mostRecentSweep && mostRecentBOS && mostRecentBOS.index > mostRecentSweep.index) {
  const oppositeDir =
    (mostRecentSweep.type === 'SSL' && mostRecentBOS.direction === 'bull') ||
    (mostRecentSweep.type === 'BSL' && mostRecentBOS.direction === 'bear');
  if (oppositeDir) return 'DISTRIBUTION';
}
// MANIPULATION: 최근 10봉 내 스윕, 후속 BOS 없음
if (mostRecentSweep && (lastIdx - mostRecentSweep.index) <= 10) return 'MANIPULATION';
// ACCUMULATION: 타이트한 범위 + 이벤트 없음
if (swingRange < avgRange * 3 && !mostRecentSweep && !mostRecentBOS) return 'ACCUMULATION';
// 기본
return 'RESET';

3단계 · 유동성 식별

renderStep3(signal) · SSL/BSL 스윕 이벤트 필터링 및 서술
스윕(Sweep) 이벤트 목록에서 현재 스윙 범위 내 항목만 추려 방향에 맞게 정렬한 뒤 최대 3개를 출력한다. 범위 외 스윕은 생략 개수만 표기한다.
getSwingRange — 스윙 범위 산출
// LONG 방향
low  = 가장 최근 확정 SSL 가격 (없으면 현재가 × 0.97)
high = 가장 최근 미확정 BSL 가격 (없으면 현재가 × 1.05)

// SHORT 방향
low  = 가장 최근 미확정 SSL 가격 (없으면 현재가 × 0.95)
high = 가장 최근 확정 BSL 가격 (없으면 현재가 × 1.03)

// NEUTRAL
low = 현재가 × 0.97, high = 현재가 × 1.03
정렬 우선순위
  1. 방향 관련성 우선 (LONG→SSL, SHORT→BSL)
  2. 시간 최신순 (time 내림차순)
  3. 최대 3개 출력, 나머지는 "외 N개 범위 외 생략"
스윕 확정 vs 미확정
상태조건의미
확정 sweep.confirmed === true 스윕 봉 이후 close로 유효성 확인 완료
미확정 sweep.confirmed === false 스윕 봉 발생했으나 아직 close 확인 대기
자동 서술 생성 조건
confirmedSSL > 0
+
direction === LONG
→
"SSL 스윕 후 반등 → 매수측 유동성 회수 시나리오"
confirmedBSL > 0
+
direction === SHORT
→
"BSL 스윕 후 하락 → 매도측 유동성 회수 시나리오"

4단계 · 핵심 증거

renderStep4(signal, gradedFVGs) · Displacement 캔들 + FVG 등급별 출력
유동성 스윕 이후 기관의 의도를 증명하는 두 가지 증거: Displacement(급격한 이동)와 FVG(Fair Value Gap)을 제시한다. FVG는 스윙 범위 내 항목만, 등급 순으로 최대 3개 출력한다.
Displacement 처리
// signal.displacements 배열의 마지막 항목만 사용
const latestDisp = disps.slice(-1)[0];

// 출력: 방향, bodyPct, 발생 시각
// 없으면: "감지되지 않음"

bodyPct: 캔들 바디가 ATR 대비 임계값 초과 여부로 결정

FVG 필터링 순서
  1. status === 'active' 만 선택
  2. 스윙 범위 안에 겹치는 것만 (fvg.high ≥ range.low && fvg.low ≤ range.high)
  3. 등급 순 정렬: A+ → A → B
  4. 최대 3개, 나머지 생략
FVG 탐지 알고리즘 (fvg.js)
// 3개 연속 캔들 c1, c2, c3 기준
// Bull FVG: c3.low > c1.high  (상승 갭 — 아래가 비어있음)
// Bear FVG: c1.low > c3.high  (하락 갭 — 위가 비어있음)

// 최소 갭 크기: (c3.low - c1.high) / c1.high >= minGapPct

// status 업데이트 (이후 캔들로 순회):
// - 'tested':    가격이 FVG 범위에 닿았으나 close가 반대편을 넘지 않음
// - 'mitigated': bull FVG에서 close < fvg.low, bear FVG에서 close > fvg.high
FVG 등급 산정 (gradeFVG 함수)
등급조건
A+ OTE 구간 내 (oteZone === 'OTE') AND MSS 직후 발생 (fvg.time >= latestMSS.time)
A OTE 구간 내 OR MSS 직후 발생 (둘 중 하나)
B 위 조건 미해당 (기본값)
OTE (Optimal Trade Entry): 스코어카드의 oteZone === 'OTE' 필드로 판단. 피보나치 62-79% 되돌림 구간.
출력 예시
## 4단계 · 핵심 증거 (Displacement + FVG) - Displacement 캔들: LTF 2026-05-15 09:00:00Z, body +3.2% (임계 통과) Active FVG (LTF, bull) *(외 1개 범위 외 생략)*: - FVG #1: $91,500.0 ~ $92,300.0 · CE $91,900.0 · status: active · **A+** (MSS 직후 + OTE 구간) - FVG #2: $90,800.0 ~ $91,200.0 · CE $91,000.0 · status: active · A

5단계 · 구조 변화 확인

renderStep5(signal) · MSS와 BOS 이벤트 필터링 및 분류
MSS(Market Structure Shift)와 BOS(Break of Structure)를 각각 스윙 범위 내 항목으로 필터링하여 출력한다. MSS는 추세 전환, BOS는 추세 연속성을 의미한다.
MSS 감지 알고리즘 (market-structure.js)
// 현재 추세 = getCurrentTrend(swings):
//   최근 스윙 4개의 HH/HL (bull) vs LH/LL (bear) 패턴

// bear 추세에서 직전 LH를 상향 돌파 → bull MSS
if (trend === 'bear' && candle.close > lastHigh.price)
  → MSS(bull)

// bull 추세에서 직전 HL을 하향 돌파 → bear MSS
if (trend === 'bull' && candle.close < lastLow.price)
  → MSS(bear)
BOS 감지 알고리즘 (market-structure.js)
// 추세 방향 무관, 직전 스윙 포인트 돌파면 BOS

// 직전 swing high 상향 돌파
if (candle.close > lastHigh.price) → BOS(bull)

// 직전 swing low 하향 돌파
if (candle.close < lastLow.price)  → BOS(bear)

// 차이: BOS는 추세 연속, MSS는 추세 전환
필터링 공통 로직 (pickTop 내부)
// 1. 스윙 범위 내 항목만 (e.price >= swingRange.low && e.price <= swingRange.high)
// 2. 방향 관련성 우선: LONG → bull MSS/BOS, SHORT → bear MSS/BOS
// 3. 시간 최신순 정렬
// 4. 최대 3개, 나머지 "외 N개 범위 외 생략"
// 5. 스윙 범위 내 항목이 0이면 "감지 없음 (범위 외 생략)" 표시
MSS vs BOS 비교
구분MSSBOS
의미 추세 전환 (반전 시그널) 추세 연속 (방향 확인)
조건 현재 추세의 반대 구조 돌파 직전 스윙 포인트 돌파 (추세 불문)
다이어리 역할 A+ FVG 등급 기준, 스토리텔링 핵심 추세 방향성 재확인
origin 기본값 LTF HTF
출력 예시
## 5단계 · 구조 변화 확인 - MSS (스윙 범위 내, 최대 3개): - (LTF) 2026-05-15 08:30:00Z · $91,000.0에서 직전 LH 돌파 → bull MSS 확정 - BOS (스윙 범위 내, 최대 3개): - (HTF) 2026-05-14 16:00:00Z · $89,500.0 → 상승 추세 연속성 확인

심화 1 · 딜링 레인지 (ERL/IRL)

renderAdvanced1(signal, erlIrl) + deriveERLIRL(signal)
NEUTRAL 시그널은 이 단계부터 심화 분석을 건너뛴다. ERL/IRL은 가격이 현재 "레인지의 어느 극단"에 있는지, 또는 "내부 유동성 구간"에 있는지를 판단한다.
ERL / IRL 개념
구분의미구현
ERL 상단 External Range Liquidity (고점) — 매수 유동성 풀 미확정 BSL 최신 or 확정 BSL 최신
ERL 하단 External Range Liquidity (저점) — 매도 유동성 풀 확정 SSL 최신 or 미확정 SSL 최신
IRL Internal Range Liquidity — 가격이 FVG/OB 내부에 있음 현재가가 activeFVG 또는 activeOB 범위 안에 있으면 해당 구간
deriveERLIRL 로직
// ERL 하단: 확정 SSL 가장 최신 (없으면 미확정 SSL)
erlLow  = confirmedSSL.slice(-1)[0] ?? pendingSSL.slice(-1)[0]

// ERL 상단: 미확정 BSL 가장 최신 (없으면 확정 BSL)
erlHigh = pendingBSL.slice(-1)[0] ?? confirmedBSL.slice(-1)[0]

// IRL: 현재가가 activeFVG 또는 activeOB 범위 내에 있는지
irlZone = activeFVGs.find(f => cp >= f.low && cp <= f.high)
       || activeOBs.find(o => cp >= o.low && cp <= o.high)
방향성 서술 조건
// LONG + ERL 상단 있음 → "ERL 상단(BSL)을 향해 이동 중 — 연료 보충 후 재진행"
// SHORT + ERL 하단 있음 → "ERL 하단(SSL)을 향해 이동 중"
// 그 외                 → "방향 결정 중"

Unicorn Setup: Breaker Block(BB)과 FVG가 가격 구간상 중첩될 때 발생하는 고확률 셋업. BB는 MSS로 반전된 이전 OB다.

for (bb of pendingBBs) {
  for (fvg of activeFVGs) {
    if (bb.low < fvg.high && bb.high > fvg.low) {
      // 겹침 구간:
      overlapLow  = Math.max(bb.low,  fvg.low)
      overlapHigh = Math.min(bb.high, fvg.high)
      ce          = (overlapLow + overlapHigh) / 2
      return { detected: true, bb, fvg, ce }
    }
  }
}
return { detected: false }
